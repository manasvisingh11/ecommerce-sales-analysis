# E-Commerce Sales Analysis

A data analysis project built to practice core data analyst skills —
data cleaning, SQL querying, EDA, and visualization.

## About the Project

I used an online sales dataset (240 transactions, 9 columns) to answer
basic business questions around revenue, product performance, and 
regional trends. The goal was to simulate the kind of analysis a 
junior data analyst would do at an e-commerce company.

## Tools Used

- Python (Pandas, Matplotlib, Seaborn) — cleaning and EDA
- MySQL — business queries and KPI extraction
- Excel — summary report

## Questions I Answered

- Which product category generates the most revenue?
- How does revenue trend month over month?
- Which region contributes the most to total sales?
- What are the overall KPIs — total revenue, orders, AOV?

## Key Numbers

- Total Revenue: $80,567.85
- Total Orders: 240
- Average Order Value: $335.70
- Total Units Sold: 518
- Product Categories: 6
- Regions: 3

## What I Learned

Working with real-world style data taught me how to handle
date formatting, groupby aggregations, and connecting SQL
results back to Python insights. The hardest part was making
sure the monthly trend chart rendered in the correct order.
